# 💸 퀀트 웹서비스, 서버 뭘 써야 할까? — 비용 vs 성능 가이드

> "배포했더니 백테스트만 돌리면 사이트가 버벅여요. 더 비싼 서버 사야 하나요?"
> 퀀트 사이트(Alpha-Helix)를 EC2에 올리고 부딪힌 **서버 선택·비용 최적화** 정리.

---

## 🩺 1. 먼저 — 왜 느린지 진단

웹사이트가 느린 원인은 보통 둘 중 하나예요:

| 느린 곳 | 원인 | 큰 서버로 해결? |
|---|---|---|
| **백테스트·ML** (vectorbt·XGBoost·SHAP·Walk-Forward) | **CPU·메모리 폭식** | ✅ 효과 큼 |
| **AI 채팅(Claude/LLM)** | 외부 API **네트워크 지연** | ❌ 거의 무관 |
| **페이지 로딩** | 정적 파일 | ❌ 이미 빠름(CDN) |
| **DB 쿼리** | 인덱스/쿼리 | 보통 무관 |

→ **퀀트의 진짜 병목은 "백테스트·ML 연산"** 이에요. 특히 Trust Score처럼 **백테스트를 수십 번 반복**하는 기능이 CPU를 다 먹어요.

### ⚠️ t3 의 함정 — "버스터블"
`t3` 계열은 **평소엔 CPU 일부만 쓰고, 잠깐만 풀파워(버스트)** 내는 타입이에요.
- 백테스트처럼 **오래 무거운 작업**을 돌리면 → **크레딧이 떨어져 느려져요(throttle)**.
- t3 기본은 "unlimited" 모드라 계속 버스트는 되지만, **추가 요금이 붙고** 그래도 2 vCPU 한계는 그대로.

---

## 🖥️ 2. 인스턴스 종류 — 한눈에

| 계열 | 성격 | 퀀트에 |
|---|---|---|
| **t3** (버스터블) | 평소 저전력, 잠깐 고성능 | 가벼운 데모용. 백테스트 많으면 ❌ |
| **m5/m6i** (범용·안정형) | CPU·메모리 균형, throttle 없음 | ⭐ 무난한 선택 |
| **c5/c6i** (연산특화) | 높은 CPU, 메모리는 적음 | 🏎️ **백테스트 속도 최우선**이면 |
| **r5** (메모리특화) | RAM 큼 | 대용량 데이터 들고 계산할 때 |

### 추천 (월 비용은 us-east 온디맨드 24h 대략치)
| 상황 | 추천 | vCPU/RAM | ~월 |
|---|---|---|---|
| 데모·소수 사용 | t3.large | 2 / 8GB | ~$60 |
| **가성비 안정형** ⭐ | **m5.large** | 2 / 8GB | ~$70 |
| 여유 있게 | m5.xlarge | 4 / 16GB | ~$140 |
| **백테스트 속도 중시** 🏎️ | **c5.xlarge / c6i.xlarge** | 4 / 8GB | ~$120 |

> 💰 **Savings Plan / 예약 인스턴스**(1년 약정)면 위 가격에서 **~30~40% 할인**. 계속 켜둘 거면 거의 필수.

### 인스턴스 유형 바꾸는 법 (데이터 유지, ~5분)
1. EC2 → 인스턴스 선택 → **인스턴스 중지**
2. **작업 → 인스턴스 설정 → 인스턴스 유형 변경** → 원하는 유형 선택
3. **인스턴스 시작**
- 도커·DB·코드 다 그대로(EBS 볼륨), **Elastic IP도 유지**. 잠깐 끊겼다 다시 떠요.

## 🧠 3. 돈 쓰기 전에 — "구조"로 푸는 게 더 싸다

서버를 키우는 건 쉽지만 매달 돈이 나가요. **같은 돈이면 구조를 바꾸는 게 더 효율적**일 때가 많아요.

### ① 백테스트를 "비동기 잡"으로
- 지금: 사용자가 백테스트 누르면 **그동안 서버가 묶임** → 다른 사람도 느려짐
- 개선: 누르면 **즉시 "계산 중…" 응답** + 뒤에서 처리 → 웹은 항상 빠릿
- (작업 큐: 간단히는 백그라운드 스레드, 크게는 Celery/RQ + Redis)

### ② 결과 캐싱
- **같은 파라미터 = 같은 결과** → 한 번 계산하면 저장해두고 재사용
- 백테스트·Trust Score는 캐싱 효과가 커요

### ③ 무거운 ML은 분리 / 스팟 인스턴스
- XGBoost 재학습 같은 배치 작업은 **별도 워커**로 빼면 웹서버가 안 묶임
- 배치는 **스팟 인스턴스**(최대 ~70% 할인) 활용 가능

### ④ 안 쓸 땐 중지 = 요금 0
- 개발/테스트 단계면 **밤에 인스턴스 중지** → 컴퓨팅 요금 0 (EBS·Elastic IP 소액만)

---

## 🎯 4. 결론 — 단계별 처방

1. **지금 당장(공짜):** 백테스트를 비동기+캐싱으로 → 체감 확 좋아짐
2. **그래도 부족하면:** `t3.medium → m5.large` (throttle 제거, 안정형) ~ $70/mo
3. **백테스트가 핵심 상품이면:** `c5/c6i.xlarge` (연산특화)로 → 백테스트 속도 최고
4. **계속 운영할 거면:** Savings Plan으로 30~40% 절감
5. **배치 ML:** 스팟 인스턴스 별도 워커

> **핵심:** "느리다 = 무조건 비싼 서버"가 아니에요. **뭐가 느린지 진단 → 구조로 먼저, 그래도 부족하면 키우기.** 퀀트는 백테스트가 병목이니 거기에 돈/구조를 쓰는 게 정답.

## 📌 우리(Alpha-Helix) 적용 메모

- 다음: 비동기 잡 + 캐싱 도입 → 그래도 부족하면 **m5.large** 또는 백테스트용 **c6i.xlarge**
- Cloudflare 프록시로 정적/HTTPS는 이미 최적화됨(서버 부하 X)
