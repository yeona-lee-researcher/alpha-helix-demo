# Alpha-Helix 실행 환경 가이드 — 입문자용 완전 매뉴얼

> "백테스트도 좋고 AI 신호도 좋다 — 그런데 **이걸 실제로 어떻게 켜고, 어떻게 KIS 증권사와 연결되며, 어떻게 안전하게 실주문까지 가나?**" 에 답합니다.

## 0. 큰 그림 한 장

```
┌──────────────────────────────────────────────────────────────────┐
│              사용자 (브라우저, https://your-domain)               │
└────────────────────────────┬─────────────────────────────────────┘
                             │ HTTPS + JWT 쿠키
                             ▼
┌──────────────────────────────────────────────────────────────────┐
│   Frontend  React 18 + Vite (:5173 dev / Nginx static prod)       │
│   - 자연어 챗으로 전략 설계 → 백테스트 → 신호 확인 → 주문 승인        │
└────────────────────────────┬─────────────────────────────────────┘
                             │ REST /api/*
                             ▼
┌──────────────────────────────────────────────────────────────────┐
│   Backend  Spring Boot 4 + Java 21 (:8080)                        │
│   - 도메인: strategy / ai / user / payment / notification          │
│   - JWT HS256, AES-GCM 키 보관, HMAC 승인 링크, Resilience4j        │
└────────┬─────────────────────┬───────────────────────┬───────────┘
         │ JDBC                │ HTTP                  │ HTTPS
         ▼                     ▼                       ▼
   ┌──────────┐    ┌─────────────────────┐    ┌────────────────┐
   │ MySQL 8  │    │ Analytics FastAPI    │    │  KIS OpenAPI   │
   │ +Flyway  │    │ (:8001) vectorbt +   │    │  (모의/실거래)  │
   └──────────┘    │  XGBoost + SHAP      │    └────────────────┘
                   └─────────────────────┘
```

3개의 독립 프로세스가 협력합니다. 하나가 죽어도 다른 것은 살아 있습니다 (Circuit Breaker).

## 1. KIS 샘플 저장소 vs Alpha-Helix — 결정적 차이

| 항목 | KIS 샘플 (`strategy_builder` + `backtester`) | **Alpha-Helix** |
|---|---|---|
| **언어 다양성** | Python + Next.js | Python(분석) + **Java/Spring(코어)** + React |
| **백테스트 엔진** | QuantConnect Lean (Docker 필수) | vectorbt (순수 Python, Docker 불필요) |
| **시장** | 한국주식 (KIS Open API 직접) | 미국 ETF 중심 (yfinance + Polygon + FRED 매크로) |
| **전략 정의** | `.kis.yaml` 비주얼 빌더 | 7개 검증된 템플릿 + AI 자연어 파라미터 추천 |
| **AI 시그널** | 없음 | **XGBoost + SHAP + Regime + Trust Score** |
| **사용자 관리** | 단일 사용자 (로컬 도구) | **회원가입/JWT/구독 플랜** (멀티 유저 SaaS) |
| **주문 안전장치** | 즉시 주문 | **MOCK 제안 → 이메일 HMAC 승인 → 실주문** (2단계) |
| **결제** | 없음 | Toss Payments (4개 플랜) |
| **배포** | 로컬 `./start.sh` | EC2 + Nginx + systemd, 환경변수 강제 |
| **데이터 출처** | KIS 시세 (실시간) | yfinance(일봉) + Polygon(분봉) + FRED(매크로) + Binance(코인) |

요약: **KIS 샘플 = 개인용 한국주식 도구**, **Alpha-Helix = 미국 ETF + AI + 멀티유저 + 안전 게이트**

## 2. 처음 실행하는 7단계 (Windows 기준)

### 2.1 사전 설치

| 도구 | 버전 | 다운로드 |
|---|---|---|
| JDK 21 | Temurin 21 | https://adoptium.net/ |
| Node.js | 20 LTS | https://nodejs.org/ |
| Python | 3.11 | https://www.python.org/ |
| MySQL | 8.x | https://dev.mysql.com/downloads/ |
| Git | 최신 | https://git-scm.com/ |

설치 확인:

```powershell
java -version    # openjdk 21
node -v          # v20.x
python --version # 3.11.x
mysql --version  # 8.x
```

### 2.2 DB 생성

```powershell
mysql -uroot -p1234 -e "CREATE DATABASE alphahelix_db CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;"
```

### 2.3 환경변수 — 필수 키만

`backend/src/main/resources/application-local.properties` 파일 생성 (템플릿: `application.properties` 복사):

```properties
# DB
DB_HOST=localhost
DB_PORT=3306
DB_NAME=alphahelix_db
DB_USERNAME=root
DB_PASSWORD=1234

# 보안 (반드시 직접 채워야 부팅됨 — 기본값 없음)
JWT_SECRET=change-me-to-random-32-bytes-or-longer-string
APP_CRYPTO_KEY=Base64로_인코딩된_정확히_32바이트_랜덤키
APPROVAL_HMAC_SECRET=another-random-secret-for-email-approval-links

# Analytics 사이드카 (FastAPI)
ANALYTICS_BASE_URL=http://localhost:8001
ANALYTICS_INTERNAL_TOKEN=any-random-string-but-must-match-fastapi

# AI (없으면 룰베이스로 폴백)
GEMINI_API_KEY=AIzaSy...

# 이메일 (Gmail 앱 비밀번호)
MAIL_USERNAME=you@gmail.com
MAIL_PASSWORD=app-password-16-chars

# 결제 (Toss 테스트 키 그대로 사용 가능)
TOSS_SECRET_KEY=test_sk_zXLkKEypNArWmo50nX3lmeaxYG5R

# 실주문 차단 (개발 중엔 true 권장)
TRADING_KILL_SWITCH=true
```

> **APP_CRYPTO_KEY 만드는 법** (PowerShell):
> ```powershell
> [Convert]::ToBase64String((1..32 | ForEach-Object { Get-Random -Maximum 256 }))
> ```

`analytics/.env` 도 생성 (`.env.example` 복사):

```env
ANALYTICS_INTERNAL_TOKEN=any-random-string-but-must-match-fastapi   # 위와 동일!
POLYGON_API_KEY=                # 선택
FRED_API_KEY=                   # 선택
DISABLE_RETRAIN_SCHEDULER=0     # 1이면 자동 재학습 중지
```

### 2.4 Backend 실행 (:8080)

```powershell
cd backend
.\gradlew bootRun --args="--spring.profiles.active=local"
```

성공 신호: 콘솔에 `Started DevBridgeApplication in X seconds`
헬스 확인: http://localhost:8080/actuator/health → `{"status":"UP"}`

### 2.5 Analytics 실행 (:8001)

새 PowerShell 창에서:

```powershell
cd analytics
python -m venv .venv
.venv\Scripts\activate
pip install -r requirements.txt
uvicorn app.main:app --port 8001 --reload
```

성공 신호: http://localhost:8001/health → `{"status":"ok"}`
Swagger UI: http://localhost:8001/docs (모든 API 클릭으로 실행)

### 2.6 Frontend 실행 (:5173)

또 새 창에서:

```powershell
cd frontend
npm install
npm run dev
```

브라우저: http://localhost:5173

### 2.7 첫 로그인 → 첫 백테스트

1. 회원가입 → 이메일 인증 (개발 환경에선 콘솔 로그에 링크 출력)
2. 로그인 → AlphaWorkspace 진입
3. **Chat 탭**: "SPY 5년 SMA 20/60 백테스트 해줘" 입력
4. **Report 탭**: QuantStats Tearsheet 확인
5. **Trust 탭**: 73점 같은 신뢰도 확인

## 3. KIS 증권사 실제 연결 (선택 단계)

### 3.1 KIS Open API 신청

1. https://apiportal.koreainvestment.com/about-howto 에서 신청
2. **모의투자 앱키/시크릿** 먼저 발급 → 충분히 테스트
3. 익숙해지면 **실전 앱키/시크릿**

### 3.2 Alpha-Helix 에 키 등록 — **AES-GCM 암호화 저장**

KIS 샘플 저장소처럼 `kis_devlp.yaml` 평문 파일이 아니라, 우리는 **DB에 암호화 저장** 합니다:

1. 프론트 → `/account/broker` → "KIS 키 등록"
2. 폼에 앱키/시크릿/HTS ID/계좌번호 입력
3. Backend `KisBrokerService` 가 `APP_CRYPTO_KEY` 로 AES-GCM 암호화 후 저장
4. 평문은 메모리에서도 즉시 소멸

### 3.3 실주문이 일어나는 흐름

```
[22:30 KST] DailySignalGenerator
    │
    ├─ Analytics /signals/today 호출 → 시그널 + XGBoost 확률 + SHAP 설명
    │
    ├─ OrderProposalService.create(MOCK)   ← 항상 MOCK 먼저
    │
    ├─ NotificationService → 이메일 전송 (HMAC 서명 링크 포함)
    │
[사용자가 메일의 "승인" 클릭]
    │
    ├─ ProposalController.approve(HMAC 검증 + TTL 확인)
    │
    ├─ TRADING_KILL_SWITCH=false 확인
    │
    └─ KisBrokerService.placeOrder() → KIS REAL API 호출
              │
              └─ 토큰 자동 갱신 + Resilience4j 재시도
```

### 3.4 5겹 안전장치

| 층 | 어디서 막나? |
|---|---|
| 1. **MOCK 우선** | `DailySignalGenerator` — 신호는 무조건 MOCK 제안으로 |
| 2. **HMAC 승인** | 메일 링크 변조 시 즉시 401 |
| 3. **TTL 만료** | `OrderProposalExpiryJob` — 시간 지나면 자동 무효화 |
| 4. **Kill Switch** | `TRADING_KILL_SWITCH=true` 한 줄로 모든 실주문 중단 |
| 5. **Rate Limit** | AI 요청 20/h, 주문 N/일 등 Bucket4j |

## 6. 운영 배포 (AWS EC2)

`deploy/DEPLOY_FROM_SCRATCH.md` 참고. 핵심:

```
EC2 (t3.small 이상 권장 — t3.micro는 OOM)
├─ /etc/nginx/sites-available/who-a   (SSL 종단 + 라우팅)
├─ systemd: who-a-backend.service     (Spring Boot)
├─ systemd: who-a-analytics.service   (FastAPI + uvicorn)
└─ MySQL 8 (RDS 또는 로컬)
```

⚠️ 주의 (메모리 [aws-ec2-deploy.md] 기록):
- t3.micro 에서 Gradle 빌드 불가 → **로컬에서 빌드 → SCP 로 jar 업로드**
- User Data 는 최초 부팅에만 실행 (stop/start 시 무시)
- Elastic IP 반드시 부착
- systemd 에 `StartLimitBurst=5` 안 걸면 부팅 실패 시 CPU 100% 데드락

## 7. 트러블슈팅 사전

| 증상 | 원인 / 해결 |
|---|---|
| Backend가 즉시 죽음 (`APP_CRYPTO_KEY required`) | 환경변수 미설정. 2.3 참고 — 기본값 없음 |
| `Analytics 502 / Circuit open` | FastAPI 가 꺼졌거나 토큰 불일치. `:8001/health` 확인 |
| `JWT signature mismatch` | `JWT_SECRET` 이 재시작마다 바뀜. 고정값으로 설정 |
| KIS `EGW00201` 초당 호출 초과 | 모의투자 제한 빡셈. 실거래 키 권장 |
| 자정에 자동 학습이 안 됨 | `DISABLE_RETRAIN_SCHEDULER=1` 인지 확인 |
| 한글 깨짐 (PowerShell) | 메모리 [powershell-encoding.md] 참고 — `Set-Content -NoNewline` 금지 |
| 이메일 미발송 | Gmail 앱 비번 (16자) 필요, 일반 비번 X |
| Toss 결제 실패 | 클라이언트키/시크릿키 **같은 계정 쌍** 확인 |
