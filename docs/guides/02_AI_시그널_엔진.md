# Alpha-Helix AI 시그널 & 신뢰도 엔진 — 입문자용 완전 가이드

> 백테스트가 "과거에 잘 됐나?" 라면, 이 문서의 엔진은 **"내일 오를까? 이 전략을 진짜 믿어도 되나?"** 를 답합니다.
> AI·통계 모두 처음이어도 따라올 수 있도록 한 줄 한 줄 풀었습니다.

---

## 0. 이 엔진이 왜 필요한가?

KIS 샘플 저장소의 `strategy_builder/`는 "지표를 조합해서 BUY/SELL 신호를 만든다" 까지만 합니다.
하지만 실전에서 가장 큰 함정 두 가지가 있습니다:

1. **과최적화(Overfitting)**: 과거 데이터에 딱 맞춘 전략은 미래에 자주 실패합니다.
2. **시장 국면 변화**: 상승장에서 잘 통한 전략이 하락장에선 전혀 안 통합니다.

Alpha-Helix는 이 두 가지를 막기 위해 백테스트 위에 **AI 시그널 + 신뢰도 점수** 를 한 겹 더 얹습니다.

```
   [규칙 기반 시그널] ───┐
   (SMA, RSI 같은 기본)  ├──→ [최종 결정]
   [XGBoost 확률]    ────┤
   (내일 오를 확률)       │
   [Regime 분석]    ─────┤   ← 지금이 상승장인지 하락장인지
   [Trust Score]    ─────┘   ← 이 전략을 얼마나 믿어도 되는지 (0~100점)
```

## 1. 4개의 핵심 모듈 한눈에

| 파일 | 역할 | 입문자용 한 줄 |
|---|---|---|
| `analytics/app/models/xgb_signal.py` | **XGBoost 분류기** — 13개 기술적 특성으로 "내일 종가가 오늘보다 오를 확률" 학습/예측 | "내일 오를 확률 67%" 식으로 답해줌 |
| `analytics/app/explain/shap_explainer.py` | **SHAP 설명** — XGBoost가 왜 그렇게 판단했는지 특성별 기여도 분해 | "RSI가 과매도라서 +12%, 거래량이 평소보다 적어서 -3%" |
| `analytics/app/robust/regime.py` | **Regime 분석** — 시장이 지금 5가지 국면 중 어디인지 판별 (HMM 또는 룰베이스) | "지금은 '고변동성 불안정' 국면이라 SMA Cross는 평소보다 30% 약함" |
| `analytics/app/robust/trust_score.py` | **Trust Score** — 워크포워드 + 국면별 + 파라미터 섭동 결과를 종합한 0~100점 | "이 전략 73점 — 실전 투입 가능" |

## 2. XGBoost 시그널 — "내일 오를 확률" 구하기

### 2.1 모델이 입력으로 받는 13개 특성

`xgb_signal.py` 의 `build_features()` 가 만드는 피처:

| 카테고리 | 피처 | 의미 |
|---|---|---|
| 수익률 | `ret_1`, `ret_5`, `ret_20` | 1일/5일/20일 수익률 |
| 추세 | `sma_20_ratio`, `sma_60_ratio`, `sma_200_ratio` | 현재가 / 이동평균 - 1 |
| 변동성 | `vol_20`, `vol_60` | 20/60일 표준편차 |
| 모멘텀 | `rsi_14`, `macd`, `macd_signal_diff` | RSI·MACD |
| 미시구조 | `range_pct`, `vol_ratio_20` | (고가-저가)/종가, 거래량 비율 |

타깃은 `y_next_up = 1 (내일 오름) / 0 (내일 떨어짐)`.

### 2.2 학습 — 자동 + 수동

- **자동**: 매일 한국시간 22:30 에 `retrain_scheduler` 가 `DEFAULT_UNIVERSE` (SPY, QQQ, TQQQ, GLD, TLT, SCHD, ...) 의 모델을 재학습
- **수동**:
  ```powershell
  curl http://localhost:8001/models/train `
    -Method POST `
    -Headers @{ "X-Internal-Token" = $token; "Content-Type" = "application/json" } `
    -Body '{ "ticker": "SPY", "period": "5y" }'
  ```
- 학습된 모델은 `analytics/models_cache/xgb_SPY.joblib` 같이 저장됩니다.

> ⚠️ **데이터 부족 시** 학습은 자동으로 스킵됩니다. 합성 데이터로 억지 학습하지 않습니다 (시장 거짓 신호 방지).

### 2.3 시계열 교차검증 (Time-Series CV)

미래 데이터로 과거를 예측하는 "데이터 누설(leakage)"을 막기 위해, 5-Fold `TimeSeriesSplit` 으로 검증합니다. 일반 K-Fold 쓰면 안 됩니다. 결과는 `cv_avg` 에 `accuracy / precision / recall` 평균이 들어옵니다.

### 2.4 사용 예 — 시그널 + 설명

```powershell
curl http://localhost:8001/signals/today `
  -Method POST `
  -Headers @{ "X-Internal-Token" = $token; "Content-Type" = "application/json" } `
  -Body '{ "tickers": ["SPY", "QQQ"], "strategy": "sma_cross", "include_ml": true }'
```

응답:

```json
{
  "signals": [
    {
      "ticker": "SPY",
      "signal": "BUY",                     ← 룰베이스 (SMA Cross)
      "ml_proba_up": 0.62,                 ← XGBoost: 62% 확률로 내일 상승
      "explanation": {
        "predicted_direction": "UP",
        "top_contributions": [
          { "feature": "sma_60_ratio", "value": 0.018, "shap": 0.041 },
          { "feature": "rsi_14",       "value": 58.2,  "shap": 0.022 },
          { "feature": "macd",         "value": 0.31,  "shap": 0.013 }
        ],
        "human_summary": "60일 이평선 위 + RSI 중립 + MACD 양수 → 상승 우위"
      }
    }
  ]
}
```

이게 매일 22:30에 자동 실행되어 **OrderProposal(주문 제안)** 으로 큐에 쌓이고, 사용자는 이메일 승인 링크(HMAC 서명)로 매수 여부를 결정합니다.

## 3. SHAP — "왜 그렇게 예측했어?"

블랙박스 AI 의 가장 큰 약점은 "이유 없음". SHAP(SHapley Additive exPlanations)은 게임이론에서 가져온 방법으로, **각 피처가 예측값을 얼마나 끌어올렸는지/내렸는지** 정확히 분해합니다.

`explain/shap_explainer.py` 의 `explain_latest()` 가 최근 시점의 예측에 대해 Top-N 기여 피처를 뽑아 줍니다. UI에선 "이 신호 왜 떴나요?" 버튼을 누르면 이 결과가 표시됩니다.

> 입문자 팁: SHAP 양수 = 상승 쪽으로 밀었음, 음수 = 하락 쪽으로 밀었음. 절댓값이 크면 중요.

## 4. Regime — "지금 시장은 어떤 모드?"

`analytics/app/robust/regime.py` 는 시장을 다음 **5개 국면** 으로 분류합니다 (v2, HMM 기반):

| 국면 | 의미 | 어울리는 전략 |
|---|---|---|
| `bull_quiet` | 잔잔한 상승 | `buy_and_hold`, `sma_cross` |
| `bull_volatile` | 들쭉날쭉 상승 | `momentum_12_1` |
| `sideways` | 횡보 | `rsi_meanrev`, `infinite_buying` |
| `bear` | 하락 | 현금 비중 ↑, `vix_risk_off` |
| `high_vol_unstable` | 공포 폭락 | 거래 중지 추천 |

### 4.1 두 가지 방법

| 방법 | 장점 | 단점 |
|---|---|---|
| `rule` (기본) | 빠르고 결정적 | 임계값에 민감 |
| `hmm` | Gaussian HMM (ret · vol20 · mom60 피처로 학습) | 표본 부족 시 fit 실패 → 자동 rule 폴백 |

응답에 `method` (실제로 어떤 방법으로 답했는지) 와 `hmm_fallback` (HMM 시도했다가 실패했는지) 가 표시됩니다.

### 4.2 호출 예

```powershell
curl http://localhost:8001/regime `
  -Method POST `
  -Headers @{ "X-Internal-Token" = $token; "Content-Type" = "application/json" } `
  -Body '{ "ticker": "SPY", "period": "5y", "strategy": "sma_cross", "method": "hmm", "smoothing": 5 }'
```

응답에는 각 국면별로 **전략의 평균 수익률 / 샤프 / 거래 수** 가 들어 있어, "이 전략은 어느 국면에서 강하고 어느 국면에서 약한가" 가 한눈에 보입니다.

## 5. Trust Score — "이 전략 얼마나 믿어도 돼?" (0~100점)

`analytics/app/robust/trust_score.py` 는 다음 4가지를 가중 평균해 한 숫자로 압축합니다:

| 구성 요소 | 비중(기본) | 설명 |
|---|---|---|
| **Walk-Forward 안정성** | 30% | 시간을 잘게 쪼개서 "학습 기간으로 만든 룰이 다음 테스트 기간에도 통하는지" 반복 검증. 통하면 점수 ↑ |
| **국면별 일관성** | 25% | 5개 국면 모두에서 평균 이상 수익? 한 국면에만 의존? 의존도가 낮을수록 점수 ↑ |
| **파라미터 섭동 강건성** | 25% | SMA 20→18, 22로 바꿔도 결과 비슷? 살짝만 바꿔도 망가지면 = 과최적화 → 점수 ↓ |
| **MDD 목표 달성** | 20% | 자산별 MDD 한도(ETF 25% / 2x 50% / 3x 75% / 개별주 35%) 안에 들어왔는지 |

### 5.1 점수 해석

| 점수 | 의미 |
|---|---|
| 80~100 | 매우 안정 — 실전 투입 가능 |
| 60~80 | 양호 — 자금 절반만 투입 권장 |
| 40~60 | 주의 — 페이퍼 트레이딩만 |
| 0~40 | 위험 — 다른 전략 고려 |

### 5.2 호출 예

```powershell
curl http://localhost:8001/trust `
  -Method POST `
  -Headers @{ "X-Internal-Token" = $token; "Content-Type" = "application/json" } `
  -Body '{ "ticker": "TQQQ", "period": "10y", "strategy": "infinite_buying" }'
```

응답:

```json
{
  "trust_score": 73,
  "breakdown": {
    "walkforward":     { "score": 78, "weight": 0.30 },
    "regime_consistency": { "score": 65, "weight": 0.25 },
    "param_robustness":   { "score": 80, "weight": 0.25 },
    "mdd_target":         { "score": 70, "weight": 0.20 }
  },
  "overfit_penalty": 7,
  "verdict": "양호 — 자금 50% 투입 권장",
  "asset_class_detected": "etf_3x",
  "mdd_target_pct": 75
}
```

## 7. AI 사용량 제어 — 안전장치

| 보호장치 | 위치 | 동작 |
|---|---|---|
| **AI Rate Limit** | `global/config/AiRateLimitFilter` | 사용자당 AI 채팅 20 req/hour (Bucket4j) |
| **LLM 폴백 체인** | `domain/ai/LlmRouter` | Gemini 실패 → OpenAI → Anthropic → Perplexity |
| **AiGateway 쿼터** | `AiGatewayService` | 일/월 사용량 DB 로그, 한도 초과 시 차단 |
| **MOCK → REAL 게이트** | `DailySignalGenerator` | 시그널은 **항상 MOCK 제안 먼저** 생성 후 사용자 승인 → 실주문 |
| **Kill Switch** | `TRADING_KILL_SWITCH=true` | KIS 어댑터가 모든 실주문 거부 |

## 8. 자주 막히는 곳

| 증상 | 원인 / 해결 |
|---|---|
| `ml_note: 모델 미학습` | XGBoost 학습 안 됨. `POST /models/train` 호출 |
| `XGBoost 학습 후에도 proba=None` | 데이터 < 200 행. `period` 를 "5y" 이상으로 |
| `HMM fit failed → rule fallback` | 표본 부족. 정상 동작, 응답의 `method` 확인 |
| Trust Score 가 매번 다름 | 파라미터 섭동에 난수가 들어감. seed 고정은 robust/trust_score 의 `np.random.seed` 수정 |
| Gemini 폴백 자꾸 발생 | `GEMINI_API_KEY` 쿼터 초과 또는 미설정 (룰베이스로 전환됨) |
