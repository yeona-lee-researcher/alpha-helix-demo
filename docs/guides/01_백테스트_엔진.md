# Alpha-Helix 백테스트 엔진 — 입문자용 완전 가이드

> "어제까지의 데이터로 내 전략이 진짜 돈을 벌었을지 시뮬레이션해 보는" 도구입니다.
> 이 문서는 **주식·코딩 모두 처음**인 분도 따라 할 수 있게 풀어 썼습니다.

---

## 0. 먼저 알아둘 5가지 용어

| 용어 | 풀어 쓰면 |
|---|---|
| **백테스트(Backtest)** | "만약 내가 작년 1월 1일에 이 전략을 시작했다면 지금 얼마가 됐을까?"를 과거 가격 데이터로 돌려보는 모의실험. |
| **티커(Ticker)** | 종목의 영문 약자. `SPY` = S&P500 ETF, `QQQ` = 나스닥 100, `005930` = 삼성전자. |
| **수수료/슬리피지(Slippage)** | 수수료는 증권사에 내는 돈, 슬리피지는 "내가 사려던 가격보다 살짝 비싸게/싸게 체결되는 손실분". 우리 엔진은 기본 **0.25% + 0.1%** 를 깐다. |
| **CAGR** | 연 평균 복리수익률. "1년에 평균 몇 % 벌었나?" |
| **MDD(Max Drawdown)** | 최고점 대비 가장 많이 깨진 비율. -30%면 1억이 7천만원까지 빠진 적 있다는 뜻. |

## 1. 우리 백테스트 엔진은 어디 있나?

```
Alpha_Helix/
└── analytics/                      ← Python 백테스트 사이드카 (포트 :8001)
    └── app/
        ├── main.py                 ← FastAPI 입구. /backtest 같은 API 정의
        ├── backtest/
        │   ├── vbt_engine.py       ← 6개 기본 전략 (vectorbt 라이브러리 사용)
        │   ├── infinite_buying.py  ← 무한매수법 (라오어식 분할매수)
        │   └── futures_engine.py   ← 선물 백테스트(실험)
        ├── data/
        │   ├── yf_client.py        ← yfinance: 미국주식/ETF 가격
        │   ├── polygon_client.py   ← Polygon: 분봉/장중 데이터
        │   ├── fred_client.py      ← 매크로(금리·실업률 등)
        │   └── binance_client.py   ← 코인 가격
        └── metrics/
            └── quantstats_report.py ← 샤프, 알파, 베타, 정보비율 계산 + HTML 리포트
```

> KIS 샘플 저장소의 `backtester/`는 **QuantConnect Lean(Docker)** 기반이라 첫 실행에 4~5GB 이미지가 다운로드됩니다.
> 우리 엔진은 **vectorbt (순수 Python)** 기반이라 `pip install`만 하면 끝 — 가볍습니다.

## 2. 엔진을 구동하는 데 필요한 것

| 항목 | 우리 버전 | (참고) KIS backtester |
|---|---|---|
| 백테스트 라이브러리 | **vectorbt 0.26** (numpy 벡터화) | QuantConnect Lean (Docker, C# 엔진) |
| Python | 3.11 | 3.11 |
| Docker | **불필요** | 필수 |
| 가격 데이터 | yfinance (무료, 일봉) + Polygon (분봉) | KIS Open API (한국주식) |
| 결과 리포트 | QuantStats HTML Tearsheet | Lean 기본 리포트 |
| 추가 분석 | XGBoost + SHAP + Trust Score | 없음 (백테스트만) |

### 2.1 처음 한 번만 하면 되는 환경 세팅

```powershell
# 1) Python 가상환경 만들기 (analytics 폴더에서)
cd analytics
python -m venv .venv
.venv\Scripts\activate            # Windows
# source .venv/bin/activate       # Mac/Linux

# 2) 라이브러리 설치 (vectorbt, xgboost, fastapi 등 한방에)
pip install -r requirements.txt

# 3) 환경변수 파일 만들기
copy .env.example .env
# .env 안의 ANALYTICS_INTERNAL_TOKEN 만 아무 문자열로 채우면 됨
```

### 2.2 서버 켜기

```powershell
uvicorn app.main:app --port 8001 --reload
```

성공하면 브라우저에서 http://localhost:8001/health 열었을 때 `{"status":"ok"}` 가 보입니다.
**Swagger 문서**: http://localhost:8001/docs (모든 API를 클릭해서 바로 실행 가능)

## 3. 우리 엔진이 지원하는 7가지 전략

| # | 전략 ID | 한 줄 설명 | 누구에게? |
|---|---|---|---|
| 1 | `buy_and_hold` | 첫날 사서 끝까지 들고 있기 | 비교 기준 (벤치마크) |
| 2 | `sma_cross` | **단기 이동평균선이 장기 이동평균선을 위로 뚫으면 매수, 아래로 뚫으면 매도** (골든/데드크로스) | 초보자 가장 추천 |
| 3 | `rsi_meanrev` | RSI(과매수/과매도 지표)가 30 밑이면 사고 70 위면 팔기 — "쌀 때 사서 비쌀 때 팔기" | 박스권 시장 |
| 4 | `macd` | MACD 선이 시그널 선 위로 가면 매수 | 추세 추종 |
| 5 | `momentum_12_1` | 최근 12개월 수익률에서 최근 1개월 수익률을 뺀 값이 양수면 매수 (학계에서 유명한 Jegadeesh-Titman 변형) | ETF 회전 |
| 6 | `vix_risk_off` | 공포지수(VIX) 25 이하일 때만 보유, 그 위면 현금화 | 방어형 |
| 7 | `infinite_buying` | **무한매수법** — 원금을 40회로 쪼개고 매일 평단보다 싸면 1회분, 살짝 비싸면 0.5회분 매수, 평단 +10% 도달 시 전량 익절 | 변동성 큰 레버리지 ETF |

> ⚠️ KIS의 `strategy_builder`처럼 80개 지표를 비주얼로 조합하는 빌더는 **현재 우리에겐 없습니다** — 대신 위 7가지를 검증된 템플릿으로 제공하고, AI 챗봇이 자연어로 파라미터를 추천해 줍니다.

## 4. 가장 쉬운 백테스트 1회 — "삼성 SPY를 SMA Cross로 5년"

API 한 번만 호출하면 됩니다. PowerShell에서:

```powershell
$token = "your-internal-token"   # .env 의 ANALYTICS_INTERNAL_TOKEN
curl http://localhost:8001/backtest `
  -Method POST `
  -Headers @{ "X-Internal-Token" = $token; "Content-Type" = "application/json" } `
  -Body '{
    "ticker": "SPY",
    "period": "5y",
    "strategy": "sma_cross",
    "sma_fast": 20,
    "sma_slow": 60,
    "initial_capital": 10000
  }'
```

응답 예시 (실제 숫자는 시점마다 다름):

```json
{
  "ticker": "SPY",
  "stats": {
    "total_return_pct": 78.4,        ← 5년 누적 +78.4%
    "cagr_pct": 12.3,                ← 연 평균 12.3%
    "max_drawdown_pct": -18.7,       ← 최악일 때 -18.7%
    "sharpe": 0.94,                  ← 0.5 이하 별로, 1 이상 양호, 2 이상 우수
    "num_trades": 14,
    "win_rate_pct": 57.1
  },
  "risk_metrics": {
    "alpha_annual": 0.018,           ← 시장(SPY) 대비 초과수익
    "beta": 0.92,                    ← 시장이 1% 빠질 때 0.92% 빠짐
    "sortino": 1.31,
    "information_ratio": 0.42
  },
  "equity_curve": [ ... ],            ← 자산 추이 (차트용)
  "report_url": "/reports/SPY_sma_cross_xxx.html"  ← HTML Tearsheet
}
```

`http://localhost:8001/reports/SPY_sma_cross_xxx.html` 를 브라우저에서 열면 **QuantStats가 만든 풀 리포트**(누적수익률, 월별 히트맵, 드로다운 차트 등 30종)가 보입니다.

## 5. 결과 지표 해석 — 입문자가 자주 헷갈리는 것

### 5.1 "총 수익률 +78% 인데 샤프가 0.5? 좋은 거야 나쁜 거야?"
- **샤프 비율(Sharpe)** 은 수익률을 변동성으로 나눈 값. 같은 수익이라도 가격이 출렁이면 점수가 깎입니다.
- 기준: `< 0.5` 별로 / `0.5 ~ 1` 보통 / `1 ~ 2` 좋음 / `> 2` 의심될 정도로 좋음 → 데이터 누락 점검 필요

### 5.2 "MDD가 -30%이면 망한 거 아냐?"
- 5년 동안 한 번이라도 -30% 까지 갔다는 뜻이지, 지금 -30%라는 게 아닙니다.
- 일반 ETF는 -20~-30%, 3배 레버리지(TQQQ 등)는 -75% 이상도 정상 범위입니다.

### 5.3 "승률 0% 인데 수익이 났어요?"
- 승률은 **이미 청산된** 매매만 셉니다. 아직 들고 있는 게 이익이면 총 수익에는 반영되지만 승률엔 안 들어갑니다.
- 즉 "승률 낮은데 수익 많음" = 미실현 이익 비중이 크다는 신호 → **실전에서 매도 조건을 다시 보세요.**

### 5.4 "CAGR이 너무 높게 나오는데?"
- 워밍업 기간(SMA 60일 같이 처음 60일은 거래 없음)도 분모에 들어가서 부풀려질 수 있습니다.
- `equity_curve` 차트에서 실제 첫 매수일이 언제인지 꼭 확인하세요.

## 6. 무한매수법 — 우리만의 특화 전략

KIS 샘플 저장소엔 없는 우리 특화 기능입니다. `analytics/app/backtest/infinite_buying.py` 의 핵심 규칙:

```
1) 원금 1만 달러를 40회분으로 나눔 → 1회분 = $250
2) 매일 종가 보고:
   - 종가 ≤ 평단가:                  $250 전액 매수  (1.0회분)
   - 평단 < 종가 ≤ 평단 × 1.15:      $125 매수      (0.5회분)
   - 그 외 (너무 비쌈):              매수 안 함
3) 종가 ≥ 평단 × 1.10 → 전량 매도 + 사이클 리셋 (수익금 다음 사이클에 복리)
```

호출 예시:

```powershell
curl http://localhost:8001/backtest/infinite-buying `
  -Method POST `
  -Headers @{ "X-Internal-Token" = $token; "Content-Type" = "application/json" } `
  -Body '{
    "tickers": ["TQQQ"],
    "period": "3y",
    "split": 40,
    "take_profit_pct": 10,
    "loc_offset_pct": 15,
    "initial_capital": 10000
  }'
```

응답에는 **각 매수일의 회차 / 매수금액 / 평단 / 사이클 종료일 / 사이클별 수익률** 까지 들어 있어, 일별 일지 형태로 검토할 수 있습니다.

## 7. 백엔드(Spring Boot)는 백테스트를 어떻게 호출하나?

```
사용자 (브라우저)
   ↓ "백테스트 시작" 클릭
Spring Boot (:8080)
   ↓ AnalyticsClient.requestBacktest()
   ↓ HTTP POST /backtest  + X-Internal-Token 헤더
FastAPI Analytics (:8001)
   ↓ vbt_engine.run_backtest()
   ↓ QuantStats compute_metrics()
응답 JSON → 백엔드가 DB 저장 → 프론트가 차트로 표시
```

- **`ANALYTICS_INTERNAL_TOKEN`** 두 프로세스가 같은 값을 공유해야 인증 통과
- Analytics 가 꺼져 있어도 Resilience4j Circuit Breaker 가 즉시 폴백 → 백엔드는 안 죽음

## 8. 자주 막히는 곳 (실전 트러블슈팅)

| 증상 | 원인 / 해결 |
|---|---|
| `ModuleNotFoundError: vectorbt` | 가상환경(.venv) 활성화 안 함. `.venv\Scripts\activate` 다시 |
| `401 invalid internal token` | `.env`의 토큰과 요청 헤더 토큰이 다름 |
| `YFinanceError: rate limit` | yfinance 무료라 분당 호출 제한. 30초 기다리거나 Polygon 키 등록 |
| 무한매수법 결과가 buy_and_hold 보다 처참 | 추세 하락장에서는 평단이 계속 깎여서 정상. **횡보·상승장**에서 강한 전략 |
| `^VIX fetch failed` | VIX 전략은 인터넷 필수. 회사망에서 막힐 수 있음 |
| HTML 리포트가 404 | `analytics/reports/` 디렉토리 권한 또는 생성 실패. 로그 확인 |
